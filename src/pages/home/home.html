Présentation

Structure d'un projet

  • Espace de développement :
    • platforms: Platform specific libraries and builds
    • plugins: Plugins added to the project
    • resources: icons and splashscreens integrated within the app
    • src: Actual app code
      • index.html: base html ppage
      • theme: CSS/SASS/LESS stylesheets
        • variable.scss
      • pages: pages (if any) code
      • app: app base code
      • assets: medias used in the app
    • www: Web-based build

Présentation

Environnement de développement

  • installer nodejs au préalable : https://nodejs.org
  • installer ionic: 
      npm install -g cordova ionic
  • créer un projet (dans le dossier courant): 
      ionic start --v2 myApp blank
  ionic start --v2 myApp tabs
  ionic start --v2 myApp sidemenu
  • créer un projet (dans le dossier courant): 
      ionic platform add ios 
  # ou
  ionic platform add android
  • lancer un mode de simulation web: 
      ionic serve
  • lancer l'application sur un device: 
      ionic run ios 
  # ou
  ionic run android

  } 
}
  // ou
var MonPrototype = {
  attribut: "valeur attribut"
}

MonPrototype.prototype = new Object();

MonPrototype.prototype.fonction = function(parametre) {
  return "valeur retour";

Concepts

Le data-binding

  • Le data-binding permet de lier une variable HTML à une variable JavaScript/TypeScript
  • Cette variable est donc accessible sur les deux langages, et peut être mis à jour d'un langage vers l'autre

Concepts

Le data-binding

  • Il existent différentes méthodes :
    • Intégration d'une variable JS dans du texte HTML : 
      <div>{{variable}}</div>
    • Exposition d'une variable à un Component : 
      <my-component [variableducomponent]="variableexterieure"></my-component>
    • Exposition d'un event d'un Component à une expression/fonction : 
      <my-component (evenement)="action()"></my-component>
    • Binding dans les deux sens (principalement réservé à NgModel) : 
      <my-component [(directive)]="variable"></my-component>

Concepts

Le data-binding

  • Pour accéder à une variable d'un Component, celle-ci doit être exposée grâce au décorateur @Input 
    import { Component, Input } from '@angular/core';

@Component({
  selector: 'user-profile',
  template: '<div>{{user.name}}</div>'
})
export class UserProfile {
  @Input() user;
  constructor() {}
}

Concepts

L'injection de dépendances

  • L'injection de dépendances consiste à pouvoir exposer un code d'un fichier/module à un autre.
    ES6 permet de faire ceci très simplement grâce aux instructions import (et export) 
    import { Component } from '@angular/core';
import { AppComponent } from './app.component';
import * from './index';
    NB: vous remarquerez que vous pouvez choisir les éléments à importer, en provenance du module concerné.
  • Pour pouvoir importer une classe d'un fichier, il faut que cette classe soit exportable : 
    export class AppComponent {}      // Cette classe est importable via 'import'

Concepts

L'injection de dépendances

  • Angular 2 vous simplifie grandement la tâche d'injection de dépendances, mais il faut quand même notifier le module de l'application de l'existence vos classes : 
    import { NgModule, ErrorHandler } from '@angular/core';
import { IonicApp, IonicModule, IonicErrorHandler } from 'ionic-angular';
import { MyApp } from './app.component';
import { HomePage } from '../pages/home/home';
import { MyService } from '../services/my-service.service';

@NgModule({
  declarations: [
    MyApp,                                                          // Composants Web utilisés dans l'application
    HomePage                                                        // Composants Web utilisés dans l'application
  ],
  imports: [
    IonicModule.forRoot(MyApp)                                      // Modules supplémentaires (ici Ionic)
  ],
  bootstrap: [IonicApp],                                            // Composant de démarrage de l'application (ici l'application Ionic)
  entryComponents: [
    MyApp,                                                          // Composants Web utilisés comme pages
    HomePage                                                        // Composants Web utilisés comme pages
  ],
  providers: [
    {provide: ErrorHandler, useClass: IonicErrorHandler},           // Les services de l'application
    MyService                                                       // Les services de l'application
  ]
})
export class AppModule {}

Concepts

Les services

  • Pour permettre d'avoir des méthodes communes à l'application, vous pouvez créer des services : 
    import { Injectable } from '@angular/core';

@Injectable()                        // Un service doit être 'Injectable'
export class AuthService {
  login() {
  }
  
  logout() {
  }
}
    NB : Un service est un singleton (= Une seule instance dans toute l'application).

Concepts

Les services

  • Une fois ajouté à l'application, vous pouvez utiliser le service où bon vous semble. 
    import { Component } from '@angular/core';
import { AuthService } from '../services/auth.service';

@Component({
  selector: 'user-profile',
  template: '<div>{{user.name}}</div>'
})
export class UserProfile {
  @Input() user;
  constructor(authService : AuthService) {
    this._authService = authService;
  }
}

Concepts

Les directives

  • Ce sont les éléments que l'ont peut appeler du code HTML
  • Il en existe trois types :
    • Les Components : des directives avec un template HTML
    • Les Directives Structurelles : des directives qui permettent de modifier le document HTML (DOM)
    • Les Directives Atrributs : des directives qui permettent de changer l'apparence ou le comportement d'un élément

Concepts

Les directives

  • Les directives structurelles :
  • Les plus communes sont
    • ngIf : équivalent de if en JS
    • ngFor : équivalent de for en JS
    • ngSwitch : équivalent de switch en JS
  • Vous pouvez aussi créer les votres.

Concepts

Les directives

  • Cas d'utilisation de directives structurelles : 
    <p *ngIf="condition">
  S'affiche si la condition est vraie.
</p>
<p *ngIf="!condition">
  S'affiche si la condition est fausse.
</p>>
  • Ceci donnera, si "condition" est vraie : 
    <p>
  S'affiche si la condition est vraie.
</p>

Concepts

Les directives

  • Lorsque le bloc de code HTML n'est pas délimité par une balise, on peut utiliser le bloc template : 
    <template [ngIf]="condition">
  <p>
    S'affiche si la condition est vraie.
  </p>
</template>
  • Ce qui donnera, si "condition" est vraie : 
    <p>
  S'affiche si la condition est vraie.
</p>

Concepts

Les directives

  • Une directive attribut, par exemple nommée myHighlight, peut s'utiliser ainsi : 
    <div myHighlight>Highlighted text</div>
  • Celle-ci peut être définie comme ceci : 
    import { Directive, ElementRef, Input } from '@angular/core';
          
@Directive({ selector: '[myHighlight]' })
export class HighlightDirective {
    constructor(el: ElementRef) { // on récupère la référence à l'élément HTML
                                  // dans lequel la directive est appelée
       // on peut faire des modifications sur cet élément ainsi
       // (ici on change sa couleur de fond pour du jaune)
       el.nativeElement.style.backgroundColor = 'yellow';
    }
}
